# 누적합
```cpp
arr = [1,2,3,4,5,6,7,8];
```
위 배열의 값을, 
```cpp
arr[0] = arr[0];
arr[1] = arr[0] + arr[1];
...
arr[n] = arr[0] + ... + arr[n];
```
이 되도록 바꿔보자.

답은
```
arr = [1,3,6,10,15,21,28,36];
```
이 된다.
즉, 특정 배열의 숫자를 더한것이다.

## 문제
만약 
arr = [1,2,3, ..., 100000] 이라는 배열이 있고, a부터 b까지의 합을 100000번 구하는 프로그램을 만들어보자.
2중 for문으로 이 문제를 해결한다면, 최악의 경우(worst case)는 100000 * 100000 = 100억 이다.. 즉 O(n^2)다..

하지만 누적합을 이용하면 더 효율적으로 문제를 풀 수 있다.

일단, psum이라는 '합 배열'이라는 것을 만들어보자

```cpp
arr = [1,2,3, ..., 100000];

int psum[100001] = {0};

psum[1] = 1;
for(int i = 2; i < 100001; i++){
    psum[i] = psum[i-1] + arr[i];
}
```
이 합배열을 가지고 a부터 b를 구해보자.

만약 2부터 4의 구간합을 구하고싶다면.

1-2 -> arr[1] + arr[2]
1-4 -> arr[1] + arr[2] + arr[3] + arr[4]
2-4 -> arr[2] + arr[3] + arr[4]
즉 a-b를 구하고 싶다면, 합배열 b번 인덱스 값에서 a-1번째 값을 빼면 된다.

```cpp
result = psum[a] - psum[b-1];
```
